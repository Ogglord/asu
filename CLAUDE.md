# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Fork of [openwrt/asu](https://github.com/openwrt/asu). Two deltas from upstream:

1. CORS middleware is always-on (`allow_origins=["*"]`) in `asu/main.py`.
2. `branches_url` setting loads branch config dynamically at runtime from an external URL (see `reload_branches()` in `asu/util.py`). This is how the fork points at ImageBuilder containers from [Ogglord/openwrt-imagebuilder-mt6000](https://github.com/Ogglord/openwrt-imagebuilder-mt6000) instead of `downloads.openwrt.org`.

When merging upstream, those two additions must survive.

## Dev workflow

Dependencies are managed by `uv`; Python 3.13+ is required (`pyproject.toml`).

```bash
uv sync --extra dev                                  # install (incl. dev extras)
uv run fastapi dev asu/main.py                       # run server (reload)
uv run rq worker                                     # run one worker
uv run pytest -vv --runslow                          # full test suite
uv run pytest tests/test_api.py::test_name -vv       # single test
uv run ruff check .                                  # lint (same as CI)
uv run ruff format --check .                         # style check (same as CI)
uv run ruff format .                                 # apply formatting
```

Slow tests (real ImageBuilder container runs) are skipped by default — use `--runslow` to include them. CI (`.github/workflows/test.yml`) runs with `--runslow` under Python 3.13 and 3.14.

Tests use `fakeredis` and `pytest-httpserver` (see `tests/conftest.py`); the fake upstream listens on `127.0.0.1:8123` and serves fixtures from `tests/upstream/`. The `asu-build` podman network and a `podman.sock` symlink must exist for slow tests — see `test.yml` for setup.

### Containerized stack (podman-compose)

```bash
podman network create asu-build                      # one-time
ln -sf /run/user/$(id -u)/podman/podman.sock podman.sock
cp asu.example.toml asu.toml                         # then edit
podman-compose up -d                                 # server + worker + redis
podman-compose -f podman-compose.yml -f podman-compose.cache.yml up -d  # + nginx cache
podman-compose -f podman-compose.workers.yml up -d   # workers-only (needs S3 in .env)
```

`asu.toml` is already present and contains local dev overrides (points at the Ogglord imagebuilder releases, redis on `:6399`, `public_path = .dev-data`). Do not commit further changes to it without checking — the comment in the file says it's generated by `dev-asu.sh`.

## Architecture

FastAPI frontend + RQ (Redis Queue) backend — the API enqueues build jobs; worker processes dequeue and spawn a disposable **podman container** per build. The Podman socket (`podman.sock`) is mounted into the worker so it can speak to the host podman daemon. Build containers join the isolated `asu-build` network (internet, but no access to Redis / host services).

**Request lifecycle** (see `asu/build.py` and `asu/routers/api.py`):

1. `POST /api/v1/build` → `BuildRequest` validated → `get_request_hash()` computes cache key.
2. If cached in Redis / store, return immediately. Otherwise enqueue via `rq`.
3. Worker picks up → `build.build()` → launches `base_container` (default `ghcr.io/openwrt/imagebuilder`, overridable via `asu.toml`) with user-requested packages → extracts outputs → uploads to store → marks Redis job done.
4. Store backend is either `LocalStore` (filesystem under `public_path/store`) or S3 (`asu/store.py`, `store_backend="s3"`).

**Branch resolution** (`asu/util.py::get_branch`) has special handling for `-SNAPSHOT` versions: it does `rsplit("-", 1)[0]` to derive the branch key. `get_container_version_tag()` strips `-SNAPSHOT` and prepends `openwrt-` to form the ImageBuilder tag. This is the contract that ties this server to the `openwrt-imagebuilder-mt6000` repo — don't change these without updating both sides.

**Config** (`asu/config.py`): `pydantic-settings` reads from `asu.toml`, environment variables, and `.env` (env wins, then TOML, then defaults). Upstream-release defaults live in `Settings.branches` but are overridden at runtime by `reload_branches()` when `branches_url` is set.

**Package changes table** (`asu/config.py::package_changes_list`): tracks OpenWrt package renames/removals across revisions. Adding an entry requires a revision number — see the top-of-file comment for how to derive it (`scripts/getver.sh yyyy-mm-dd` in an OpenWrt checkout).

**Stats** (`asu/routers/stats.py`, `asu/util.py::add_timestamp`): optional (`server_stats = true` in `asu.toml`), requires Redis Stack with the TimeSeries module (the default `redis-stack-server` image has it). Build-event names are asserted in `add_build_event()`.
